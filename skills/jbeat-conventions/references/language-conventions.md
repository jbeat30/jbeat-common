# Language Conventions

## Default Language: Korean

- Write all code comments in Korean unless the project requires another language.
- Write all task result reports and validation summaries in Korean by default.
- Use another language only when the user explicitly requests it.

## When to Use Another Language

- User explicitly requests a different language for this session or task.
- Project local rules specify a different language.
- External system documentation or contracts use a specific language that must be preserved.

## Scope

This language policy applies to:
- Inline comments (`//`)
- JSDoc comments (`/** ... */`)
- Final result reports
- Validation summaries
- Status and progress messages to the user

It does not apply to:
- Code identifiers (variable names, function names, class names) — follow naming conventions
- External API contracts, field names, or specifications
- File names and directory names
